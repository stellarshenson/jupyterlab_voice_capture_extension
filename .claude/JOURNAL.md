2. **Task [Extended] - Implement voice-capture pipeline** (v0.1.1): Built the full browser-to-FIFO pipeline against the acceptance criteria in `docs/acc-crit-voice-capture.md`<br>
   **Result**: Replaced the hello-world scaffold with the complete capture chain. Frontend: `src/voice-capture.ts` owns the state machine (idle/streaming/error via a Lumino signal), `getUserMedia`, an `AudioContext({sampleRate:16000})` so the browser resamples natively (B1), reconnect with capped backoff (D1), secure-context/permission/no-device error mapping (E1-E4), and full track + context teardown (A3/D2). `src/worklet.ts` ships the AudioWorklet processor as an inline Blob URL (no extra static asset) - it downmixes to mono and emits fixed 320-sample s16le frames off the UI thread (B2/B3/D4). `src/pcm.ts` holds the pure `floatToS16LE` + `Framer` helpers (Jest, 100% covered). `src/status.ts` adds a status-bar mic icon that pulses faint red while streaming (`@keyframes vc-pulse`); `src/index.ts` wires the palette command + status item with optional tokens so activation always logs. Server: `routes.py` `VoiceCaptureWebSocketHandler(WebSocketMixin, WebSocketHandler, JupyterHandler)` enforces auth via `@ws_authenticated` (C1) and does last-writer takeover (D3); `sink.py` `FifoSink` runs a background writer thread + bounded queue, creates the FIFO if absent, refuses a non-FIFO path (C4/C5), and tolerates an absent reader without blocking the IOLoop (C3); `__init__.py` exposes a `VoiceCapture` traitlet for the configurable sink path. Chose last-writer takeover and the native-resample path per user decision. Build hardening: bumped `typescript` to ~5.8 and added webpack/chalk `overrides`+`resolutions` (this builder turned out to use Rspack, so unnecessary but harmless). Verified end-to-end: `make install` builds the wheel and both extension lists report OK, Jest 5/5, pytest 3/3, and `jupyterlab.browser_check` activates the extension. The C1 unauthenticated-rejection test became a structural assertion on the `__allow_unauthenticated` flag because a raw HTTP/ws client deadlocks on the in-process test IOLoop.
